Anteproyecto - Liher Ríos Ruiz 1. TÍTULO DEL PROYECTO El título de este proyecto va a ser "Implementación de automatizaciones para entornos educativos". 2. INTRODUCCIÓN DEL PROYECTO En este proyecto pretendo dar solución a problemas cotidianos por medio de las automatizaciones. A todos nos aburre hacer una tarea repetitiva una y otra vez, sea con la frecuencia que sea; es por ello, que he pensado en utilizar la herramienta n8n para "solo tener que pensar una vez." Una vez la automatización está creada, la tarea debería ser completada en un solo click. 3. FINALIDAD La finalidad de este proyecto es optimizar el trabajo a la hora de tener que hacer tareas sencillas que demandan una gran cantidad de tiempo, tratando de poder resolver de forma indefinida la misma tarea una vez su solución esté creada. 4. OBJETIVOS Los objetivos de este proyecto; al estar enfocado a los entornos y sistemas educativos, es "hacer la vida más fácil" a aquellos usuarios que trabajen en un colegio o academia. Además, mi solución permite tener una herramienta portátil y ligera para poder trabajar en cualquier lugar sin necesidad de descargas. 5. MEDIOS HARDWARE Y SOFTWARE A UTILIZAR Para llevar a cabo este proyecto, voy a disponer de las siguientes herramientas. En cuanto a hardware, voy a utilizar un USB en el que poder alojar el Docker que vamos a utilizar para ejecutar la herramienta. No obstante, voy a crear un repositorio en GitHub en donde subiré los archivos que se necesitan para poner el proyecto en marcha. Por la parte del software, vamos a utilizar varias herramientas. Docker es la herramienta que hace de cimientos, puesto que utilizo un contenedor compuesto con Docker Compose para poder tener persistencia en el contenedor y que los cambios se guarden. Para ello utilizo una BBDD SQL Lite para poder guardar en una ruta específica todos nuestros workflows. Además, utilizamos la herramienta n8n como motor para generar automatizaciones que nos den soluciones a los problemas que nos encontremos por el camino. Todas las herramientas corren dentro de un mismo contenedor. 6. PLANIFICACIÓN En principio el proyecto no debería ser muy complicado de llevar a cabo. De hecho el setup inicial no debería llevarme más allá de unas horas. El punto fuerte de mi proyecto es que es muy escalable. Puedes hacer tantas automatizaciones como quieras, que eso es lo que verdaderamente lleva tiempo y trabajo. Lo bueno que todo ese tiempo que inviertas, solo deberías emplearlo una vez. Hecha la automatización, problema resuelto para siempre.
